# Model Candidates

This document tracks the first model candidates for independent validation.

## ASR First

### faster-whisper

Role: general ASR baseline.

Why:

- Easy Python integration.
- Good Mandarin and English baseline.
- Supports CPU and CUDA.

Recommended comparison order:

| Model | Local path | Expected use |
| --- | --- | --- |
| `small` | `models/asr/faster-whisper-small` | Fast baseline and CPU-friendly validation. |
| `medium` | `models/asr/faster-whisper-medium` | Better accuracy, recommended next step on GPU. |
| `large-v3` | `models/asr/faster-whisper-large-v3` | Best Whisper-family quality candidate, slower and heavier. |

Run:

```bash
python auralis_lab/asr.py --engine faster_whisper --model models/asr/faster-whisper-small --audio samples/asr/Tencent_test.wav
python auralis_lab/asr.py --engine faster_whisper --model models/asr/faster-whisper-medium --audio samples/asr/Tencent_test.wav
python auralis_lab/asr.py --engine faster_whisper --model models/asr/faster-whisper-large-v3 --audio samples/asr/Tencent_test.wav
```

Notes:

- Auralis converts Chinese ASR output to simplified Chinese by default with OpenCC.
- Punctuation is generated by the ASR model itself. The CLI provides an initial prompt, but this is not a hard guarantee.

### FunASR Paraformer + Punctuation

Role: Mandarin-focused ASR candidate.

Why:

- Strong Chinese ASR ecosystem.
- Punctuation can be added with a dedicated FunASR punctuation model.

Local paths:

```text
models/asr/funasr-paraformer-zh
models/asr/funasr-ct-punc
```

Run:

```bash
python auralis_lab/asr.py --engine funasr --model models/asr/funasr-paraformer-zh --punc-model models/asr/funasr-ct-punc --audio samples/asr/Tencent_test.wav
```

### SenseVoiceSmall

Role: Chinese/multilingual speech understanding candidate.

Why:

- Good Mandarin interaction candidate.
- Can output readable text with punctuation in our current tests.
- The main practical public candidate is `SenseVoiceSmall`.

Local path:

```text
models/asr/sensevoice-small
```

Run:

```bash
python auralis_lab/asr.py --engine sensevoice --model models/asr/sensevoice-small --audio samples/asr/Tencent_test.wav
```

### sherpa-onnx

Role: ONNX Runtime deployment candidate.

Why:

- Useful for PC-side and CPU-friendly deployment experiments.
- `sherpa-onnx` is a runtime/framework, not one ASR model.
- Test specific ONNX models through it, such as SenseVoice, Paraformer, or Zipformer CTC.

Recommended first ONNX model path:

```text
models/asr/sherpa-onnx-sense-voice-zh-en-ja-ko-yue-2024-07-17
```

Run:

```bash
python auralis_lab/asr.py --engine sherpa_onnx --sherpa-model-type sensevoice --model models/asr/sherpa-onnx-sense-voice-zh-en-ja-ko-yue-2024-07-17 --audio samples/asr/Tencent_test.wav
```

Other supported CLI modes:

```bash
python auralis_lab/asr.py --engine sherpa_onnx --sherpa-model-type paraformer --model models/asr/<sherpa-onnx-paraformer-model-dir> --audio samples/asr/Tencent_test.wav
python auralis_lab/asr.py --engine sherpa_onnx --sherpa-model-type zipformer_ctc --model models/asr/<sherpa-onnx-zipformer-ctc-model-dir> --audio samples/asr/Tencent_test.wav
```

### Piper

Role: lightweight local TTS baseline.

### CosyVoice

Role: preferred higher-quality Mandarin TTS candidate.

Validated modes:

- SFT with `CosyVoice-300M-SFT`
- zero-shot with `CosyVoice2-0.5B`
- instruct2 with `CosyVoice2-0.5B`

### VoxCPM

Role: high-quality local TTS candidate with zero-shot voice cloning.

Validated model families:

- `VoxCPM-0.5B`
- `VoxCPM2`

### ChatTTS

Role: exploratory conversational TTS candidate.

Current decision:

- Not selected for the main Auralis TTS path.
- Its dependency changes disturbed the validated CosyVoice2 environment during our tests.
- Prefer a separate conda environment if it is tested again later.

## LLM Third

### Ollama + Qwen3-8B

Role: first local LLM validation path.

Setup:

```bash
ollama pull qwen3:8b
ollama serve
```

Run:

```bash
python auralis_lab/llm.py --engine ollama --model qwen3:8b --thinking off --prompt "请用一句话介绍你自己。"
```

Use non-thinking mode by default for voice interaction so the answer stays short and suitable for TTS. Keep Qwen2.5-7B-Instruct as a fallback baseline only if Qwen3-8B is too slow or unstable on the target GPU.
